Создание event (события)

Endpoint: https://backend.calendaria.kz/events/create_event/

Метод: POST

Входные данные: {"title": "test", "all_day": true, "start_date": "28-02-2023 02:12:11" , "end_date": "28-02-2023 02:12:11", "notification_setting":"2400", "repeat":"Every Day", "event_type":"Work", "location_name" : "Almaty", "lng":40.41, "lat": 40.41, "description":"some text", "cover_image":File or null}

Ответ: {"event_id": 10}

Ошибка: {message: "failed"}

Результат: создастся event и вернется event_id который можно использовать для следующих endpoint

Описание: Предназначен для создание event принадлежащий пользователю

Добавление ссылок к event

Endpoint: https://backend.calendaria.kz/events/add_links/

Метод: POST

Входные данные: { "event_id": 10, "links":["https://stackoverflow.com/questions/14212527/how-to-set-default-value-to-the-inputtype-date", "http://localhost:63342/calendaria-backend/docs/events.html?_ijt=l4och8tc0hf6c2koln7ddeqnvd&_ij_reload=RELOAD_ON_SAVE"] }

Ответ: {"message":"links are saved"}

Ошибка: { "non_field_errors": [ "учетные данные не были предоставлены" ] }

Результат: Cсылки event будут сохранены

Описание: к event будут добавлены ссылки и будут доступны для дальнейшего использования

Добавление файлов к event

Endpoint: https://backend.calendaria.kz/events/add_attachments/

Метод: POST

Входные данные: { "event_id": 10, "attachments":[{"attachment_type": "File", "attachment": File}] }

Ответ: {"message":"attachments are saved"}

Ошибка: { "non_field_errors": [ "учетные данные не были предоставлены" ] }

Результат: файлы event будут сохранены

Описание: к event будут добавлены файлы и будут доступны для дальнейшего использования

Приглашение пользователей в event

Endpoint: https://backend.calendaria.kz/events/invite_users/

Метод: POST

Входные данные: { "event_id": 10, "participants":[{"user_id": 1, "user_permission": "admin"}] }

Ответ: {"message":"users invited"}

Ошибка: { "non_field_errors": [ "учетные данные не были предоставлены" ] }

Результат: пользователи будут приглашены в event

Описание: к event будут привязаны индивидуальные модели приглашений и занесены в базу для дальнейшего использования

ответ пользователя на приглашение в event

Endpoint: https://backend.calendaria.kz/events/user_response/

Метод: POST

Входные данные: { "invite_id": 1, "answer": "denied" }

Ответ: {"message":"invite updated"}

Ошибка: { "non_field_errors": [ "учетные данные не были предоставлены" ] }

Результат: пользователь сможет принять и отклонить свое приглашение в event

получить event

Endpoint: https://backend.calendaria.kz/events/get_opened_event/:int/

Метод: GET

Ответ: {все данные по event}

Ошибка: { {"message":"event do not exist}

Результат: получение всех данных по одному ивенту
